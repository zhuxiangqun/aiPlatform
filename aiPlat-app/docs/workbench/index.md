# Workbench 模块

> Workbench 模块是 AI Platform 应用层的 Web 工作台，提供可视化交互界面，用于调试、测试和监控。

---

## 一、模块定位

### 1.1 核心职责

Workbench 模块在整个 AI Platform 架构中承担以下核心职责：

| 职责 | 说明 |
|------|------|
| **Web 界面** | 提供 Web 管理界面 |
| **交互式对话** | 可视化 AI 对话 |
| **调试工具** | 请求调试和响应查看 |
| **监控面板** | 服务状态和指标监控 |
| **日志查看** | 实时日志查看 |

### 1.2 与相邻模块的关系

```
┌─────────────────────────────────────────────────────────────┐
│                    Web Browser                             │
└────────────────────────────┬────────────────────────────────┘

┌─────────────────────────────────────────────────────────────┐
│                  Workbench UI                              │
│  ┌─────────────┐  ┌─────────────┐  ┌─────────────────────┐  │
│  │  Chat UI   │  │  Debug UI │  │  Monitor UI        │  │
│  │  (对话)    │  │  (调试)  │  │  (监控)           │  │
│  └─────────────┘  └─────────────┘  └─────────────────────┘  │
└────────────────────────────┬────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────┐
│                    API Gateway                             │
└────────────────────────────┬────────────────────────────────┘
                             │
                             ▼
┌─────────────────────────────────────────────────────────────┐
│             Core Runtime（aiPlat-core）                     │
└─────────────────────────────────────────────────────────────┘
```

> 说明：此处的 “Core Runtime” 指 **aiPlat-core（Layer 1）** 的执行引擎（Harness/编排/Agents/Skills）；Workbench 只通过 platform API 调用，不承载执行逻辑。

## 三、配置结构

### 3.1 Workbench 配置

```yaml
# aiPlat-app/workbench/config.yaml

workbench:
  # ==================== 服务配置 ====================
  server:
    host: "0.0.0.0"
    port: 3000
    debug: false
    
  # ==================== 认证配置 ====================
  auth:
    enabled: true
    username: "admin"
    password: "${WORKBENCH_PASSWORD}"
    
  # ==================== 功能配置 ====================
  features:
    chat: true
    debug: true
    monitor: true
    logs: true
    config: true
    
  # ==================== 主题配置 ====================
  theme:
    mode: "light"           # light/dark
    primary_color: "#2563eb"
```

## 四、API 接口

### 4.1 管理接口

| 端点 | 方法 | 说明 |
|------|------|------|
| `/workbench/` | GET | 主页面 |
| `/workbench/health` | GET | 健康检查 |
| `/workbench/api/chat` | POST | 对话接口 |
| `/workbench/api/debug` | POST | 调试接口 |
| `/workbench/api/status` | GET | 状态接口 |
| `/workbench/api/logs` | GET | 日志接口 |

## 五、使用示例

### 5.1 启动 Workbench

```bash
# 启动服务
aiplat workbench

# 指定端口
aiplat workbench --port 3000

# 开发模式
aiplat workbench --debug
```

### 5.2 访问界面

```bash
# 浏览器访问
http://localhost:3000

# 认证登录
Username: admin
Password: ${WORKBENCH_PASSWORD}
```

### 5.3 使用 Chat 功能

1. 输入用户消息
2. 选择模型和参数
3. 发送请求
4. 查看响应

### 5.4 使用 Debug 功能

1. 构建请求 JSON
2. 发送测试请求
3. 查看响应
4. 保存历史请求

## 六、安全设计

### 6.1 认证

1. **基础认证**：HTTP Basic Auth
2. **会话管理**：Cookie 会话
3. **密码强度**：强制密码复杂度

### 6.2 权限

1. **只读模式**：可配置只读
2. **IP 白名单**：限制访问 IP

## 七、与旧系统差异

### 7.1 架构差异

| 方面 | 旧系统 (RANGEN) | 新系统 (aiPlat-app) |
|------|----------------|-------------------|
| 界面风格 | 基础 HTML | React/Vue |
| 功能 | 有限 | 完整 |
