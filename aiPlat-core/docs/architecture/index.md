## 二、Harness 作为操作系统的体现

### 2.1 操作系统类比

```
┌────────────────────────────────────────────────────────────────┐
│                       通用操作系统                              │
├──────────────┬──────────────┬──────────────┬─────────────────┤
│   进程管理   │   调度器     │   系统监控   │   文件系统     │
│   Process    │   Scheduler  │   Monitor   │   FileSystem   │
└──────────────┴──────────────┴──────────────┴─────────────────┘
         │              │              │              │
         ▼              ▼              ▼              ▼
┌────────────────────────────────────────────────────────────────┐
│                       Harness 操作系统                          │
├──────────────┬──────────────┬──────────────┬─────────────────┤
│  Agent生命周期│  执行循环    │  观察系统    │   记忆系统     │
│   lifecycle  │    loop      │observability│    memory      │
├──────────────┴──────────────┴──────────────┴─────────────────┤
│                         │              │                       │
│                    协调系统        权限系统                    │
│                  coordination    permissions                   │
└────────────────────────────────────────────────────────────────┘
```

### 2.2 核心能力映射

| OS 概念 | Harness 实现 | 说明 |
|---------|-------------|------|
| **进程** | Agent 实例 | 独立的执行单元 |
| **线程** | Task | Agent 中的任务 |
| **调度器** | Execution Loop | 任务调度和执行 |
| **文件系统** | Memory/Knowledge | 数据存储和检索 |
| **设备驱动** | Tools | 外部能力扩展 |
| **系统监控** | Observability | 状态监控和告警 |
| **权限控制** | Permissions | 安全和权限管理 |
| **进程间通信** | Coordination | Agent 协作 |
| **系统调用** | Hooks | 扩展点拦截 |

## 三、LangGraph 与 LangChain 的体现

### 3.1 LangChain 位置

```
harness/
└── infrastructure/
    └── langchain/          ← LangChain 集成位置
        ├── models.py       ← ChatOpenAI, ChatAnthropic
        ├── memory.py       ← ConversationMemory
        ├── tools.py        ← FunctionTool
        └── prompts.py      ← PromptTemplate
```

### 3.2 LangGraph 位置

```
harness/
└── execution/
    └── langgraph/          ← LangGraph 集成位置
        ├── graphs/         ← 图定义
        │   ├── tri_agent.py    ← TriAgent 图
        │   ├── react.py        ← ReAct 图
        │   └── multi_agent.py ← MultiAgent 图
        ├── nodes/           ← 节点定义
        │   ├── registry.py    ← 节点注册表
        │   ├── reason_node.py ← 推理节点
        │   ├── act_node.py    ← 行动节点
        │   └── observe_node.py← 观察节点
        ├── executor.py       ← 图执行器
        ├── core.py          ← 核心功能
        └── callbacks.py     ← 回调
```

## 四、数据流示例

### 4.1 Agent 执行流程

```
用户请求
    │
    ▼
┌──────────────────────────────────────────────────────────────────┐
│ apps/agents/ReActAgent.execute(context)                        │
└─────────────────────────────────┬────────────────────────────────┘
                                  │
                                  ▼
┌──────────────────────────────────────────────────────────────────┐
│ harness/interfaces/IAgent.execute()                             │
└─────────────────────────────────┬────────────────────────────────┘
                                  │ delegates to
                                  ▼
┌──────────────────────────────────────────────────────────────────┐
│ harness/execution/loop.py                                       │
│                                                                  │
│  while should_continue(state):                                  │
│      │                                                           │
│      ▼ 1. Reasoning                                             │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │ harness/execution/langgraph/nodes/reason_node.py       │    │
│  │   → uses LangChain: model.generate(prompt)              │    │
│  └──────────────────────────────────────────────────────────┘    │
│      │                                                           │
│      ▼ 2. Acting (if needed)                                    │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │ harness/execution/langgraph/nodes/act_node.py            │    │
│  │   → calls tools via LangChain                             │    │
│  └──────────────────────────────────────────────────────────┘    │
│      │                                                           │
│      ▼ 3. Observing                                             │
│  ┌──────────────────────────────────────────────────────────┐    │
│  │ harness/execution/langgraph/nodes/observe_node.py        │    │
│  │   → processes results                                     │    │
│  └──────────────────────────────────────────────────────────┘    │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### 4.2 LangChain 到 LangGraph 的集成

LangChain 提供基础能力，LangGraph 实现图编排，Harness 在 execution/langgraph/ 中整合二者：

- LangChain 基础能力：模型调用、工具定义、记忆管理
- LangGraph 图结构：状态流编排、节点定义、边连接
- Harness 集成：封装为可配置的 ReAct 执行图
