# 总览模块

> 平台健康状态总览、告警中心、快速导航入口

---

## 一、模块定位

总览模块是 AI 平台管理系统的入口，提供：

- **四层架构健康状态总览**
- **资源使用概览**
- **告警摘要与处理**
- **快速导航入口**

---

## 四、适配器层

总览模块通过适配器层连接各层：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         总览模块                                    │
├─────────────────────────────────────────────────────────────────────┤
│                                                                     │
│  ┌─────────────────────────────────────────────────────────────┐   │
│  │                    DashboardAggregator                        │   │
│  │                    (聚合各层状态数据)                          │   │
│  └─────────────────────────────────────────────────────────────┘   │
│                              │                                      │
│         ┌────────────────────┼────────────────────┐                │
│         │                    │                    │                │
│         ▼                    ▼                    ▼                │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐           │
│  │InfraAdapter │    │CoreAdapter  │    │PlatformAdapter│   ...     │
│  │(基础设施)   │    │(核心能力)   │    │(平台服务)   │           │
│  └─────────────┘    └─────────────┘    └─────────────┘           │
│         │                    │                    │                │
│         ▼                    ▼                    ▼                │
│  ┌─────────────┐    ┌─────────────┐    ┌─────────────┐           │
│  │aiPlat-infra │    │aiPlat-core │    │aiPlat-platform│   ...      │
│  │(Layer 0)   │    │(Layer 1)   │    │(Layer 2)   │           │
│  └─────────────┘    └─────────────┘    └─────────────┘           │
│                                                                     │
└─────────────────────────────────────────────────────────────────────┘
```

**适配器职责**：

| 适配器 | 连接层级 | 职责 |
|--------|----------|------|
| InfraAdapter | Layer 0 | 获取基础设施层健康状态、资源指标 |
| CoreAdapter | Layer 1 | 获取核心能力层健康状态、Agent/Skill状态 |
| PlatformAdapter | Layer 2 | 获取平台服务层健康状态、API指标 |
| AppAdapter | Layer 3 | 获取应用接入层健康状态、渠道状态 |

## 五、实现文件

### 前端

| 文件 | 功能 |
|------|------|
| `frontend/src/pages/Overview/Overview.tsx` | 平台总览页面 |
| `frontend/src/pages/Alerts/Alerts.tsx` | 告警中心页面 |
| `frontend/src/services/overviewApi.ts` | 总览 API 服务 |
| `frontend/src/services/alertsApi.ts` | 告警 API 服务 |

### 后端

| 文件 | 功能 |
|------|------|
| `management/api/dashboard.py` | Dashboard API 路由 |
| `management/api/alerts.py` | 告警 API 路由 |
| `management/dashboard/aggregator.py` | 状态聚合器 |
| `management/dashboard/infra_adapter.py` | 基础设施层适配器（真实数据） |
| `management/dashboard/core_adapter.py` | 核心能力层适配器 |
| `management/dashboard/platform_adapter.py` | 平台服务层适配器 |
| `management/dashboard/app_adapter.py` | 应用接入层适配器 |
| `management/dashboard/service_detector.py` | 服务检测器（检测PostgreSQL/Redis/Milvus/RabbitMQ/Ollama等） |
| `management/dashboard/system_info.py` | 系统信息采集（CPU/GPU/内存/存储） |

**InfraAdapter 真实数据检测**：

```
┌─────────────────────────────────────────────────────────────────────┐
│                         InfraAdapter                                 │
├─────────────────────────────────────────────────────────────────────┤
│                                                                      │
│  ┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐  │
│  │  PostgreSQL     │    │  Redis          │    │  Milvus        │  │
│  │  端口检测+连接  │    │  端口检测+INFO   │    │  端口检测       │  │
│  └─────────────────┘    └─────────────────┘    └─────────────────┘  │
│                                                                      │
│  ┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐  │
│  │  RabbitMQ       │    │  Ollama         │    │  Network        │  │
│  │  端口检测       │    │  端口检测+模型  │    │  psutil连接统计 │  │
│  └─────────────────┘    └─────────────────┘    └─────────────────┘  │
│                                                                      │
│  ┌─────────────────┐    ┌─────────────────┐                         │
│  │  Storage        │    │  Memory/GPU     │                         │
│  │  shutil磁盘统计 │    │  psutil系统信息 │                         │
│  └─────────────────┘    └─────────────────┘                         │
│                                                                      │
└─────────────────────────────────────────────────────────────────────┘
```

**健康状态计算**：

各组件根据实际运行状态返回：
- `healthy`: 服务正常运行
- `degraded`: 服务部分功能受限（如GPU未检测到）
- `unhealthy`: 服务未运行或不可用

Dashboard分数基于所有组件状态计算平均值（healthy=100, degraded=50, unhealthy=0）。

## 六、配置说明

### 健康检查配置

```yaml
# config/management.yaml
management:
  layers:
    infra:
      endpoint: "http://localhost:8001"
      enabled: true
      health_check_interval: 60
    core:
      endpoint: "http://localhost:8002"
      enabled: true
    platform:
      endpoint: "http://localhost:8003"
      enabled: false
    app:
      endpoint: "http://localhost:8004"
      enabled: false
```

### 告警规则配置

```yaml
# config/alerts.yaml
alerts:
  rules:
    - name: gpu_temperature_high
      condition: "gpu_temperature > 85"
      severity: critical
      duration: 60
      
    - name: memory_usage_high
      condition: "memory_usage > 90"
      severity: warning
      duration: 300
      
    - name: service_pending
      condition: "service_ready_replicas < service_replicas"
      severity: warning
      duration: 120
```
