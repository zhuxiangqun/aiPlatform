# 服务管理

> AI 推理服务的完整生命周期管理

---

## 一、模块定位

服务管理负责 AI 推理服务（如 LLM、向向量数据库等）的部署、配置、扩缩容和运维。

### 核心职责

```
服务管理
├── 服务生命周期
│   ├── 部署新服务（选择镜像、配置资源）
│   ├── 服务扩缩容（手动/自动）
│   ├── 服务重启/停止
│   ├── 服务删除
│   └── 服务配置更新
│
├── 服务监控
│   ├── 服务状态（运行/部署中/异常）
│   ├── Pod实例状态
│   ├── 资源使用（GPU/CPU/内存）
│   └── 服务日志/事件
│
└── 镜像管理
    ├── 镜像列表
    ├── 版本管理
    └── CI/CD集成
```

## 三、API 端点

### 服务管理

```python
# 获取服务列表
GET /api/infra/services
Query: namespace, status, type

# 获取服务详情
GET /api/infra/services/{service_id}

# 部署新服务
POST /api/infra/services
Body: {
  "name": "gpt4-api",
  "image": "vllm/vllm-openai:latest",
  "replicas": 4,
  "gpu_count": 4,
  "config": {
    "max_tokens": 4096,
    "temperature": 0.7
  }
}

# 更新服务配置
PUT /api/infra/services/{service_id}
Body: {
  "config": {
    "max_tokens": 8192
  }
}

# 扩缩容
POST /api/infra/services/{service_id}/scale
Body: {
  "replicas": 8
}

# 重启服务
POST /api/infra/services/{service_id}/restart

# 停止服务
POST /api/infra/services/{service_id}/stop

# 删除服务
DELETE /api/infra/services/{service_id}

# 获取服务日志
GET /api/infra/services/{service_id}/logs
Query: lines, follow

# 获取服务事件
GET /api/infra/services/{service_id}/events
```
