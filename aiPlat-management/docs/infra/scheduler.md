# 算力调度

> GPU 资源调度与任务队列管理

---

## 一、模块定位

算力调度负责 GPU 资源的分配策略、任务队列管理、弹性伸缩等功能。

### 核心职责

```
算力调度
├── 资源配额
│   ├── 配额管理（按团队/用户）
│   ├── 配额分配
│   ├── 配额限制
│   └── 配额使用统计
│
├── 调度策略
│   ├── 调度策略配置
│   ├── 优先级管理
│   ├── 节点选择策略
│   └── 资源亲和性
│
├── 任务队列
│   ├── 任务提交
│   ├── 任务排队
│   ├── 任务执行
│   └── 任务历史
│
└── 弹性伸缩
    ├── 自动扩缩容
    ├── 扩缩容策略
    ├── 触发条件配置
    └── 扩缩容历史
```

## 三、API 端点

### 资源配额

```python
# 获取配额列表
GET /api/infra/quotas
Query: team, status

# 获取配额详情
GET /api/infra/quotas/{quota_id}

# 创建配额
POST /api/infra/quotas
Body: {
  "name": "prod-team",
  "gpuQuota": 8,
  "team": "production"
}

# 更新配额
PUT /api/infra/quotas/{quota_id}
Body: {
  "gpuQuota": 16
}

# 删除配额
DELETE /api/infra/quotas/{quota_id}
```

### 调度策略

```python
# 获取策略列表
GET /api/infra/scheduling/policies

# 创建策略
POST /api/infra/scheduling/policies
Body: {
  "name": "high-priority",
  "priority": 100,
  "nodeSelector": {"gpu": "true"},
  "affinity": {...}
}

# 更新策略
PUT /api/infra/scheduling/policies/{policy_id}

# 删除策略
DELETE /api/infra/scheduling/policies/{policy_id}
```

### 任务队列

```python
# 获取任务列表
GET /api/infra/queue/tasks
Query: queue, status

# 提交任务
POST /api/infra/queue/tasks
Body: {
  "name": "train-llm",
  "gpu_count": 8,
  "gpu_type": "A100",
  "queue": "high",
  "priority": 80,
  "config": {...}
}

# 取消任务
POST /api/infra/queue/tasks/{task_id}/cancel

# 调整优先级
PUT /api/infra/queue/tasks/{task_id}/priority
Body: {"priority": 90}
```

### 弹性伸缩

```python
# 获取扩缩容策略列表
GET /api/infra/autoscaling/policies

# 创建扩缩容策略
POST /api/infra/autoscaling/policies
Body: {
  "service": "gpt4-api",
  "minReplicas": 2,
  "maxReplicas": 10,
  "metrics": [
    {"type": "cpu", "target": 80},
    {"type": "gpu", "target": 85}
  ]
}

# 更新策略
PUT /api/infra/autoscaling/policies/{policy_id}

# 暂停扩缩容
POST /api/infra/autoscaling/policies/{policy_id}/pause

# 恢复扩缩容
POST /api/infra/autoscaling/policies/{policy_id}/resume

# 获取扩缩容历史
GET /api/infra/autoscaling/history
```
