# 知识库管理

> 知识库与文档的完整管理

---

## 一、模块定位

知识库管理负责知识库的创建、文档管理、向量索引和检索优化。

### 核心职责

```
知识库管理
├── 知识库管理
│   ├── 创建/删除知识库
│   ├── 知识库配置
│   ├── 知识库状态监控
│   └── 知识库统计分析
│
├── 文档管理
│   ├── 文档上传
│   ├── 文档解析
│   ├── 文档分块
│   └── 文档状态跟踪
│
├── 向量索引管理
│   ├── 索引创建
│   ├── 索引更新
│   ├── 索引重建
│   └── 索引状态监控
│
└── 检索管理
    ├── 检索测试
    ├── 检索优化
    ├── 相似度配置
    └── 检索日志
```

### 知识类型

| 类型 | 说明 | 支持格式 |
|------|------|----------|
| **文档知识** | PDF、Word、Markdown | .pdf, .docx, .md |
| **代码知识** | 代码库、API文档 | .py, .js, .java |
| **网页知识** | 网页内容、在线文档 | URL |
| **结构化知识** | 数据库、表格、知识图谱 | .csv, .json, .sql |

## 三、API 端点

### 3.1 知识库管理

```python
# 获取知识库列表
GET /api/core/knowledge/collections
Query: ?status=active&limit=100&offset=0
Response: {
  "collections": [...],
  "total": 12
}

# 创建知识库
POST /api/core/knowledge/collections
Body: {
  "name": "产品文档",
  "description": "产品相关文档",
  "config": {
    "embedding_model": "text-embedding-ada-002",
    "dimension": 1536,
    "chunk_size": 512,
    "chunk_overlap": 50,
    "similarity_threshold": 0.75
  }
}
Response: {
  "id": "kb-001",
  "status": "created"
}

# 获取知识库详情
GET /api/core/knowledge/collections/{collection_id}

# 更新知识库配置
PUT /api/core/knowledge/collections/{collection_id}
Body: {
  "config": {
    "similarity_threshold": 0.8
  }
}

# 删除知识库
DELETE /api/core/knowledge/collections/{collection_id}

# 重建索引
POST /api/core/knowledge/collections/{collection_id}/reindex
```

### 3.2 文档管理

```python
# 上传文档
POST /api/core/knowledge/documents
Body: multipart/form-data {
  "collection_id": "kb-001",
  "file": "<binary>",
  "metadata": {
    "category": "product",
    "language": "zh"
  }
}
Response: {
  "document_id": "doc-001",
  "status": "parsing"
}

# 获取文档状态
GET /api/core/knowledge/documents/{document_id}
Response: {
  "id": "doc-001",
  "collection_id": "kb-001",
  "name": "产品手册.pdf",
  "status": "indexed",
  "chunks": 128,
  "size_mb": 2.3,
  "created_at": "2026-04-01T10:00:00Z"
}

# 获取文档列表
GET /api/core/knowledge/collections/{collection_id}/documents

# 删除文档
DELETE /api/core/knowledge/documents/{document_id}
```

### 3.3 检索管理

```python
# 检索测试
POST /api/core/knowledge/search
Body: {
  "collection_id": "kb-001",
  "query": "如何使用产品进行数据分析？",
  "top_k": 5,
  "similarity_threshold": 0.75,
  "filters": {
    "category": "product"
  }

Response: {
  "results": [
    {
      "content": "产品提供强大的数据分析功能...",

        "document_id": "doc-001",
        "page": 15,

      }
    }
  ],
  "duration_ms": 45
}

# 获取检索日志
GET /api/core/knowledge/collections/{collection_id}/search/logs?limit=100
```

## 四、数据模型

```typescript
// 知识库定义
interface KnowledgeCollection {
  id: string;
  name: string;
  description: string;
  status: 'active' | 'indexing' | 'error';
  config: CollectionConfig;
  document_count: number;
  total_size_mb: number;
  created_at: string;
  updated_at: string;
}

// 知识库配置
interface CollectionConfig {
  embedding_model: string;
  dimension: number;
  chunk_size: number;
  chunk_overlap: number;
  similarity_threshold: number;
}

// 文档定义
interface Document {
  id: string;
  collection_id: string;
  name: string;
  type: string;
  size_mb: number;
  status: 'parsing' | 'chunking' | 'embedding' | 'indexed' | 'error';
  chunks: number;
  metadata: Record<string, any>;
  created_at: string;
  indexed_at?: string;
}

// 检索结果
interface SearchResult {
  content: string;
  score: number;
  metadata: {
    document_id: string;
    page?: number;
    source: string;
    chunk_id?: string;
  };
}

// 索引状态
interface IndexStatus {
  collection_id: string;
  status: 'ready' | 'indexing' | 'error';
  progress: number;
  documents_indexed: number;
  total_documents: number;
  last_indexed_at: string;
}
```
