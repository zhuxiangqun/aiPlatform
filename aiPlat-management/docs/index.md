# AI平台管理系统文档

> 独立的运维管理系统 - 横切四层架构的统一管理平面

---

## 一、系统定位

AI平台管理系统是 AI Platform 的独立运维平台，提供从基础设施到应用的全生命周期管理。

### 核心价值

- **完整运维能力**：从节点到服务的全生命周期管理
- **统一管理视图**：四层架构的健康状态总览
- **深度基础设施管理**：节点、服务、算力、存储、网络的完整运维
- **成本与效率优化**：资源追踪、调度优化、自动化运维

## 九、架构边界

**重要**：理解 aiPlat-management 和 aiPlat-infra 的架构边界至关重要。

**架构图**：
```
┌─────────────────────────────────────────────────────────────────────┐
│                      Frontend (Web UI)                              │
│                           端口 5173                                 │
└────────────────────────────────┬────────────────────────────────────┘
                                 │ HTTP (浏览器)
                                 ▼
┌─────────────────────────────────────────────────────────────────────┐
│                  aiPlat-management (8000)                            │
│                       管理系统层                                    │
│                                                                     │
│  职责：Dashboard聚合、Alerting告警、Diagnostics诊断                  │
│  API: /api/dashboard/*, /api/alerting/*, /api/diagnostics/*        │
│       /api/infra/* (HTTP 转发到 infra 层)                            │
│                                                                     │
│  调用方式：HTTP 客户端 (httpx)                                      │
└────────────────────────────────┬────────────────────────────────────┘
                                 │
                                 │ HTTP 调用
                                 ▼
┌─────────────────────────────────────────────────────────────────────┐
│                     aiPlat-infra (8001)                             │
│                       基础设施业务层                                │
│                                                                     │
│  职责：实现基础设施管理业务逻辑，提供 REST API                      │
│  API: /api/infra/nodes, /api/infra/services, ...                   │
│                                                                     │
│  Manager: NodeManager, ServiceManager, StorageManager, ...         │
└─────────────────────────────────────────────────────────────────────┘
```

### 9.1 层级职责

| 层级 | 端口 | 职责 | 调用方式 |
|------|------|------|----------|
| **aiPlat-management** | 8000 | 管理系统，聚合展示 | HTTP 调用 infra API |
| **aiPlat-infra** | 8001 | 基础设施业务实现 | 直接调用 Manager 类 |

### 9.2 启动顺序

**必须按顺序启动：**

```bash
# 1. 先启动 aiPlat-infra
cd aiPlat-infra && ./start.sh

# 2. 再启动 aiPlat-management
cd aiPlat-management && ./start.sh
```
