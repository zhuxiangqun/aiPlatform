# deployment 模块（Platform Layer 2：部署控制面）

> 本文档为**骨架**：用于补齐文档引用与边界说明，后续可按实现细节逐步完善。

## 重要说明：部署“执行面”不在 platform

为避免与分层架构冲突，需要明确：

- **部署/编排的执行面**（K8s/Helm/脚本/资源操作）属于 **aiPlat-infra / 运维体系（Layer 0 + Ops）**
- platform 的 `deployment`（若存在）应当仅承担 **控制面 API**：权限校验、策略校验、任务下发、状态聚合

## 1. 定位与职责（控制面）

- 对外提供部署任务 API（创建/回滚/查询状态）
- 权限、租户隔离、审计
- 部署策略参数校验（蓝绿/金丝雀/滚动）
- 聚合来自 infra/执行器的状态（只读视图）

## 2. 边界

- ❌ platform 不直接操作基础设施资源（不直接调用 `aiPlat-infra`）
- ✅ 如需触发部署执行：通过 core 的抽象能力或专用执行器接口（由 infra/ops 提供）
