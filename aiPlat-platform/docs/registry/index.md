# registry 模块（Platform Layer 2）

> 本文档为**骨架**：用于补齐文档引用与边界说明，后续可按实现细节逐步完善。

## 1. 定位与职责

`registry` 用于管理平台层对外暴露的“能力目录/资源目录”，典型包括：

- Agent/Skill/Tool 等能力条目在平台层的注册与索引（**仅元数据与权限控制**）
- 版本/可用性状态聚合（只读）
- 租户维度的可见性控制

## 2. 边界

- 能力的真实实现与执行在 `aiPlat-core`
- registry 不应承载执行逻辑，仅做“目录/索引/治理”
