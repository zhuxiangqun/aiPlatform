# aiPlat-infra 运维指南

## 概述

本文档为运维人员提供 aiPlat-infra 基础设施层的运维指导，帮助运维人员理解基础设施层的部署配置、监控告警和故障排查。

## 部署配置

### 环境要求

- Python 3.10 或更高版本
- PostgreSQL 13 或更高版本
- Redis 6 或更高版本
- Milvus 2.0 或更高版本（可选）

### 配置文件

基础设施层使用 YAML 格式的配置文件，主要配置项包括：

- 数据库连接配置
- Redis 连接配置
- 向量数据库配置
- LLM提供商配置
- 日志配置
- 监控配置

### 环境变量

基础设施层支持通过环境变量覆盖配置：

- 数据库连接字符串
- Redis 连接字符串
- API 密钥
- 日志级别
- 调试模式

### 部署方式

基础设施层支持以下部署方式：

- **容器部署**：使用 Docker 容器部署
- **Kubernetes 部署**：使用 Kubernetes 编排部署
- **虚拟机部署**：在虚拟机上直接部署

## 算力管理运维

### 资源监控

**关键指标**：
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| GPU 利用率 | GPU 使用百分比 | > 90% 持续 5 分钟 |
| GPU 显存 | 显存使用量 | > 85% |
| CPU 使用率 | CPU 使用百分比 | > 80% |
| 任务队列 | 等待调度的任务数 | > 100 |

### 故障处理

**GPU 不可用**：
1. 检查 nvidia-smi 状态
2. 检查 CUDA 驱动版本
3. 检查 GPU 节点状态

**任务调度失败**：
1. 检查资源配额
2. 检查调度器状态
3. 查看任务日志

## 故障排查

### 常见问题

#### 数据库连接失败

**现象**：数据库连接超时或失败

**排查步骤**：

1. 检查数据库服务是否运行
2. 检查网络连接是否正常
3. 检查连接配置是否正确
4. 检查连接池是否耗尽

#### 缓存连接失败

**现象**：Redis 连接超时或失败

**排查步骤**：

1. 检查 Redis 服务是否运行
2. 检查网络连接是否正常
3. 检查连接配置是否正确
4. 检查内存是否耗尽

#### 向量数据库查询慢

**现象**：向量查询延迟高

**排查步骤**：

1. 检查索引配置是否正确
2. 检查向量维度是否合适
3. 检查内存是否足够
4. 检查查询参数是否合理

#### LLM 调用失败

**现象**：LLM 调用返回错误

**排查步骤**：

1. 检查 API 密钥是否有效
2. 检查网络连接是否正常
3. 检查配额是否用尽
4. 检查错误日志

### 故障处理流程

1. **发现问题**：通过监控或用户反馈发现问题
2. **确认问题**：检查日志和指标确认问题
3. **定位问题**：分析日志和追踪定位问题
4. **解决问题**：修复问题或临时规避
5. **复盘总结**：总结经验，防止再次发生

### 应急预案

#### 服务不可用

1. 检查服务状态
2. 重启服务
3. 回滚到上一版本
4. 通知相关人员

#### 数据丢失

1. 停止写入操作
2. 检查备份状态
3. 恢复数据
4. 验证数据完整性

#### 性能下降

1. 检查资源使用情况
2. 扩容或优化
3. 限流或降级
4. 通知相关人员
